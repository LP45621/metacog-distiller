# Set-and-forget 偏好

【元认知含义】偏好'配置一次，永远运行'的系统模式。这是一种深层的认知偏好——希望系统在配置完成后不再需要认知负担。

【证据链】'后台挂一个''隔一段时间就自己整理'→automation每4小时运行→双automation并行。本质是对认知资源的最优化分配。
